OMA Estimations: 권장 사항 산정 방식

OMA Estimations는 complexity estimation과 time estimation logic을 설명합니다. OS type, disk size, storage migration, post-migration check 시간을 기준으로 계획 수립에 필요한 근거를 제공합니다.

  • 적용 버전: V0.7.0
  • 마지막 업데이트: 12-03-2026

Complexity Estimation

복잡도는 OS type과 disk size라는 두 가지 기준으로 나뉩니다. score는 planning과 우선순위 지정에 활용되며 시간 계산에 직접 반영되는 값은 아닙니다.

Score의미
0OS 인식 불가
1비교적 단순한 migration
2일반적인 migration
3복잡한 migration
4수동 개입 가능성 높음

Disk Size Score

  • 0-10TB: Score 1
  • 11-20TB: Score 2
  • 21-50TB: Score 3
  • 50TB 초과: Score 4

Time Estimation

Storage Migration calculator와 Post-Migration Checks calculator를 결합하여 전체 migration duration을 산정합니다.

Storage Migration

기본 transfer rate는 620 Mbps로, 500GB당 약 110분 baseline과 일치합니다. 실제 환경에서는 Migration Forecaster 측정값을 사용하면 더 현실적인 산정이 가능합니다.

Post-Migration Checks

VM count, VM당 troubleshooting minutes, engineer 수, 하루 작업 시간을 기반으로 post-migration 확인 시간을 계산합니다.

이 문서는 원문 문서를 기반으로 한 한국어 번역·정리본입니다.
